This program is extremely simple and you don't really need to use PHP to create a page like this. The only thing it shows is: Hello world using the PHP echo statement. Note that the file does not need to be executable or special in any way. The server recognizes that this file needs to be interpreted by PHP due to the use of the ".php" extension, since the server is configured to send it to PHP. Think of it as a normal HTML file that has a number of special tags available that you can do a lot of cool things with.


If you tried to use this example and it did not produce any results, you were asked if you wanted to download the file, or the entire file was displayed as text, then most likely PHP is not enabled on your server or configured properly. Ask your administrator to enable it using the Installation chapter of the manual. If you are working locally, also read the installation chapter to make sure everything is set up properly. Make sure you are accessing the file via http and the server is displaying the result. If you're opening the file from the filesystem, it probably won't be parsed by PHP. If the problem persists, feel free to use one of the many options in » PHP Support.


The focus of this example is the formatting of PHP special tags. In this example we use to indicate the start of a PHP tag. Then we put the statement and leave PHP mode by adding the closing tag ?>. This way, you can go in and out of PHP mode in an HTML file whenever you want. For more information, read the section of the manual titled Basic PHP Syntax.


Note: A note about line feeds


Line feeds make little sense in HTML, but it's still a good idea to make your HTML code look clean and clear by putting line feeds. PHP will automatically remove line feeds after a closing ?> tag. This can be very useful when putting a lot of PHP blocks or including files containing PHP that are not supposed to display anything. At the same time, it can be a bit confusing. You can put a space after the closing ?> tag to force a space and line feed, or you can put an explicit line feed in the last echo/print within the PHP block.

Note: A note about word processors


Word processors like StarOffice Writer, Microsoft Word and Abiword are not good options for editing PHP files. If you want to use one of these programs to test this script, make sure to save the document as plain text, or else PHP won't be able to read and execute it.


Note: A note about Windows Notepad


If you write your PHP scripts using Windows Notepad, you must ensure that your files are saved with the .php extension. (Notepad automatically adds the .txt extension to files unless you follow the steps below to prevent it.) When you save the file and the program asks what name you want to give the file, enclose the name in quotes (ie "hello.php"). An alternative is to click the "Text Documents (*.txt)" drop-down menu in the "Save As" dialog, and change to the "All Files (*.*)" option. Here you can write the name of the file without the quotes.
