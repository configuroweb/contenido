Today, JavaScript can run not only in browsers, but also on servers or even on any device that has a special program called the JavaScript engine or interpreter.


The browser has an embedded engine sometimes called a "JavaScript Virtual Machine."


Different engines have different "code names". For example:


V8 – in Chrome, Opera and Edge.

SpiderMonkey – in Firefox.

…There are other codenames such as “Chakra” for IE, “JavaScriptCore”, “Nitro” and “SquirrelFish” for Safari, etc.

These terms are good to remember because they are used in developer articles on the internet. We will also use them. For example, if "feature X is supported by V8", then it probably works in Chrome, Opera, and Edge.


What can JavaScript do in the browser?

Modern JavaScript is a "secure" programming language. It does not provide low-level access to memory or the CPU (CPU), as it was initially created for browsers which do not require it.


The capabilities of JavaScript are highly dependent on the environment in which it runs. For example, Node.JS supports functions that allow JavaScript to arbitrarily read and write files, make network requests, and so on.


In the JavaScript browser you can do anything related to the manipulation of a web page, interaction with the user and the web server.


For example, in the browser JavaScript is capable of:


Add new HTML to the page, change existing content, and modify styles.

React to user actions, run on mouse clicks, pointer movements, and key presses.

Send network requests to remote servers, download and upload files (Technologies called AJAX and COMET).

Get and set cookies, ask the visitor questions and display messages.

Remember data on the client side with local storage.
