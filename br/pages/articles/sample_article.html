Hoje, o JavaScript pode ser executado não apenas em navegadores, mas também em servidores ou mesmo em qualquer dispositivo que possua um programa especial chamado mecanismo ou interpretador JavaScript.


O navegador possui um mecanismo incorporado às vezes chamado de "Máquina Virtual JavaScript".


Diferentes motores têm diferentes "nomes de código". Por exemplo:


V8 – no Chrome, Opera e Edge.

SpiderMonkey – no Firefox.

…Existem outros codinomes como “Chakra” para IE, “JavaScriptCore”, “Nitro” e “SquirrelFish” para Safari, etc.

É bom lembrar esses termos porque são usados ​​em artigos de desenvolvedores na Internet. Também os usaremos. Por exemplo, se o "recurso X for suportado pelo V8", provavelmente funcionará no Chrome, Opera e Edge.


O que o JavaScript pode fazer no navegador?

O JavaScript moderno é uma linguagem de programação "segura". Ele não fornece acesso de baixo nível à memória ou à CPU (CPU), pois foi criado inicialmente para navegadores que não o exigem.


Os recursos do JavaScript são altamente dependentes do ambiente em que ele é executado. Por exemplo, o Node.JS oferece suporte a funções que permitem que o JavaScript leia e grave arquivos arbitrariamente, faça solicitações de rede e assim por diante.


No navegador JavaScript você pode fazer qualquer coisa relacionada à manipulação de uma página web, interação com o usuário e o servidor web.


Por exemplo, no navegador JavaScript é capaz de:


Adicione novo HTML à página, altere o conteúdo existente e modifique estilos.

Reaja às ações do usuário, execute cliques do mouse, movimentos do ponteiro e pressionamentos de teclas.

Envie solicitações de rede para servidores remotos, faça download e upload de arquivos (Tecnologias chamadas AJAX e COMET).

Obtenha e defina cookies, faça perguntas ao visitante e exiba mensagens.

Lembre-se de dados no lado do cliente com armazenamento local.
