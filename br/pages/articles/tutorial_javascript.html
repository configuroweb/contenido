Para executar o código JavaScript que você pretende executar no back-end, ele precisa ser interpretado e, bem, executado. É isso que o Node.js faz, usando a máquina virtual V8 do Google, o mesmo ambiente de tempo de execução para JavaScript que O Google Chrome usa.


Além disso, o Node.js vem com muitos módulos úteis, então você não precisa escrever tudo do zero, como algo que gera uma string para o console.


Portanto, o Node.js é realmente duas coisas: um ambiente de tempo de execução e uma biblioteca.


Para fazer uso deles (a biblioteca e o ambiente), você precisa instalar o Node.js. Em vez de repetir o processo aqui, visite as instruções oficiais de instalação. Volte quando tiver sua versão do Node.js em execução.


"Olá mundo"

OK. Então, vamos pular na água fria e escrever nosso primeiro aplicativo Node.js: "Hello World".


Abra seu editor favorito e crie um arquivo chamado helloworld.js. Queremos escrever "Hello World" para STDOUT, e aqui está o código necessário para fazer isso:


console.log("Olá Mundo");

Salve o arquivo e execute-o por meio do Node.js:


nó helloworld.js

Isso deve retornar Hello World no seu monitor.

Casos de uso

Vamos mantê-lo simples, mas realista:


O Usuário deve poder ocupar nosso aplicativo com um navegador.

O usuário deve ver uma página de boas-vindas ao solicitar http://domain/home, que exibe um formulário de upload.

Ao escolher um arquivo de imagem para fazer upload e enviar o formulário, a imagem deve ser enviada para http://domain/upload, onde será exibida assim que o upload for concluído.

Muito bem. Agora, você pode conseguir esse objetivo pesquisando e programando qualquer coisa, mas não é isso que queremos fazer aqui.


Mais do que isso, não queremos apenas escrever o código mais básico possível para atingir esse objetivo, por mais elegante e correto que seja esse código. Adicionamos intencionalmente mais abstração do que o necessário para ter uma ideia de como é construir aplicativos Node.js mais complexos.
