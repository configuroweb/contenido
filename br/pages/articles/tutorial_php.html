Este programa é extremamente simples e você realmente não precisa usar PHP para criar uma página como esta. A única coisa que mostra é: Hello world usando a instrução PHP echo. Observe que o arquivo não precisa ser executável ou especial de forma alguma. O servidor reconhece que este arquivo precisa ser interpretado pelo PHP devido ao uso da extensão ".php", pois o servidor está configurado para enviá-lo ao PHP. Pense nisso como um arquivo HTML normal que tem várias tags especiais disponíveis com as quais você pode fazer muitas coisas legais.


Se você tentou usar este exemplo e não produziu nenhum resultado, foi perguntado se deseja baixar o arquivo ou se o arquivo inteiro foi exibido como texto, provavelmente o PHP não está ativado em seu servidor ou configurado corretamente. Peça ao seu administrador para habilitá-lo usando o capítulo Instalação do manual. Se você estiver trabalhando localmente, leia também o capítulo de instalação para certificar-se de que tudo está configurado corretamente. Verifique se você está acessando o arquivo via http e o servidor está exibindo o resultado. Se você estiver abrindo o arquivo a partir do sistema de arquivos, ele provavelmente não será analisado pelo PHP. Se o problema persistir, sinta-se à vontade para usar uma das muitas opções em » Suporte PHP.


O foco deste exemplo é a formatação de tags especiais PHP. Neste exemplo usamos para indicar o início de uma tag PHP. Então colocamos a declaração e saímos do modo PHP adicionando a tag de fechamento ?>. Dessa forma, você pode entrar e sair do modo PHP em um arquivo HTML sempre que quiser. Para mais informações, leia a seção do manual intitulada Sintaxe PHP Básica.


Observação: uma observação sobre feeds de linha


Os feeds de linha fazem pouco sentido em HTML, mas ainda é uma boa ideia fazer com que seu código HTML pareça limpo e claro colocando feeds de linha. O PHP removerá automaticamente os feeds de linha após uma tag de fechamento ?>. Isso pode ser muito útil ao colocar muitos blocos PHP ou incluir arquivos contendo PHP que não deveriam exibir nada. Ao mesmo tempo, pode ser um pouco confuso. Você pode colocar um espaço após a tag de fechamento ?> ​​para forçar um espaço e alimentação de linha, ou você pode colocar uma alimentação de linha explícita no último echo/print dentro do bloco PHP.

Existem muitos editores de texto e ambientes de desenvolvimento integrados (IDEs) que podem ser usados ​​para criar, editar e gerenciar arquivos PHP. Nota: Uma nota sobre processadores de texto


Processadores de texto como StarOffice Writer, Microsoft Word e Abiword não são boas opções para editar arquivos PHP. Se você quiser usar um desses programas para testar este script, certifique-se de salvar o documento como texto simples, ou então o PHP não poderá lê-lo e executá-lo.


Nota: Uma nota sobre o Bloco de Notas do Windows


Se você escrever seus scripts PHP usando o Bloco de Notas do Windows, deverá garantir que seus arquivos sejam salvos com a extensão .php. (O Bloco de Notas adiciona automaticamente a extensão .txt aos arquivos, a menos que você siga as etapas abaixo para evitar isso.) Quando você salvar o arquivo e o programa perguntar qual nome você deseja dar ao arquivo, coloque o nome entre aspas (ou seja, "hello.php"). Uma alternativa é clicar no menu suspenso "Documentos de texto (*.txt)" na caixa de diálogo "Salvar como" e mudar para a opção "Todos os arquivos (*.*)". Aqui você pode escrever o nome do arquivo sem as aspas.
