Mas como vamos fazer isso? Na década de 1950, quando os computadores eram enormes e consumiam muita energia, você convertia código de montagem em código de máquina correspondente em 1 e 0 usando folhas de mapeamento. Posteriormente, este código será inserido nos cartões da máquina e enviado ao computador. O computador lerá esses códigos e executará o programa. Este seria um processo longo até que a ASSEMBLER viesse ajudar.


O que são Assembler e Compiler?

Com o avanço da tecnologia à medida que os dispositivos de E/S foram inventados, você pode escrever seu programa diretamente no PC usando um programa chamado ASSEMBLER. Ele o converte para o código de máquina correspondente (110001 ..) e o envia para seu processador. Voltando ao nosso exemplo de adição de (1 + 2), o montador converterá esse código em código de máquina e saída.




Além disso, você também terá que fazer chamadas para criar funções fornecidas pelo sistema operacional para exibir a saída do código.


Mas a montadora sozinha não está envolvida em todo esse processo; ele também requer que o compilador compile o código longo em um pequeno trecho de código. Com o avanço nas linguagens de desenvolvimento de software, esse conjunto de código completo pôde ser reduzido a uma única linha de impressão f 1 + 2 A com a ajuda de um software chamado COMPILER. Ele é usado para converter seu código da linguagem C em código assembly, e o montador o converte no código de máquina correspondente, e esse código de máquina será passado para o processador. O processador mais comum usado em PCs ou computadores é o processador Intel.
